Essentials Datalag & event-struktur
Grunnmuren. Uten dette fungerer ingenting nedover i kjeden.
✓
Google Enhanced Ecommerce er implementert som standard
Plattformen pusher standardiserte events til dataLayer: view_item, add_to_cart, begin_checkout, purchase med korrekte felter.
✓
item_id i dataLayer matcher feed-ID eksakt
Den samme produkt-IDen brukes i dataLayer og i alle produktfeeder. Verifiser ved å sammenligne ecommerce.items[0].item_id mot <g:id> i feeden.
✓
GTM er installert korrekt på alle sider
GTM-snippet er på alle sider inkludert checkout og ordrebekreftelse. Verifiser med GTM Preview eller Tag Assistant.
✓
GTM-variabler leser fra ecommerce.items (ikke egne felt)
Alle piksler bruker samme dataLayer-variabel som kilde. Ingen egne felt som fb_items eller tiktok_items — disse populeres sjelden korrekt.
✓
ViewContent, AddToCart og Purchase fyrer med content_ids
Alle tre events sender content_ids, content_type: product, value og currency. Verifiser i Meta Pixel Helper under "Custom parameters".
✓
Purchase-eventet fyrer kun én gang per ordre
Kjøp må ikke telles dobbelt. Sjekk ordrebekreftelsessiden og bekreft at Purchase-eventet ikke fyrer ved reload.
Viktig Server-side & deduplication
Nødvendig for pålitelig datakvalitet og GDPR-compliance.
✓
Conversions API (CAPI) / server-side er satt opp
Server-side tracking via Stape.io, Elevar eller tilsvarende. Sender events direkte til Meta og Google uten avhengighet av nettleser.
✓
Deduplication er korrekt satt opp
Browser og server sender identisk event_id per event slik at Meta og Google ikke teller samme event dobbelt.
✓
Consent Mode v2 er implementert
Google Consent Mode v2 er aktiv med korrekt CMP-integrasjon. Modelerte konverteringer aktiveres for brukere uten samtykke.
✓
Enhanced Matching er aktivert (Meta)
Sender hashet e-post, telefon og navn via piksel eller CAPI for å forbedre match rate. Krever brukerens samtykke.
✓
Funnel-events er komplette (AddToCart + BeginCheckout)
Google Ads og Meta trenger funnel-data for optimal PMax og Advantage+ optimalisering. Mangler disse fyrer algoritmen delvis blind.
✓
Én piksel per katalog — ikke flere piksler på samme katalog
Flere piksler koblet til samme katalog splitter match rate. Én piksel per katalog. Bruk piksel-deling mellom Business Managers ved behov.
✓
Konverteringsverdier er faktisk kjøpsverdi
Verifiser at value i Purchase-eventet reflekterer faktisk ordresum inkl. eventuell rabatt — ikke listepris.
Avansert Optimering & fremtidssikring
Gir konkurransefortrinn og bedre algoritme-optimalisering over tid.
✓
Market-parameter sendes med alle events
For flerspråklige butikker: send market: "NO" / "SE" / "INT" med alle events for korrekt katalog-matching per marked.
✓
Variant-ID sendes ved kjøp (størrelse/farge)
Send variant-spesifikk ID ved AddToCart og Purchase slik at Meta og Google vet hvilken variant som faktisk ble kjøpt — ikke bare produktgruppen.
✓
Offline konverteringer er koblet til (CAPI for fysisk handel)
Relevante for brands med fysiske butikker. Kobler offline kjøp til digitale annonseklikk via ordre-ID matching.
✓
GTM-beholderen er dokumentert og ryddig
Tagger, triggere og variabler har beskrivende navn. Utdaterte tagger er arkivert. Én ansvarlig part eier GTM-kontoen.

Essentials Obligatoriske felter
Må være på plass for at produkter godkjennes og annonser kan kjøres.
✓
id matcher item_id i dataLayer eksakt
<g:id> i feeden og item_id fra dataLayer er identiske. Dette er den hyppigste årsaken til 0% katalog-match.
✓
title er beskrivende og søkevennlig
Inkluder: merke + produktnavn + nøkkelattributt. Maks 150 tegn. Unngå salgsfraser ("kjøp nå", "billig"). Eksempel: "Alfa Glimmer Dame Tursko GTX".
✓
description er ren tekst uten HTML eller BBcode
Ingen <span>, [b] eller andre formateringskoder. Ren prosa på 500–5000 tegn med naturlige søkeord.
✓
link peker til korrekt domene
Produkt-URL i feeden matcher domenet der pikselen fyrer. Mismatch mellom feed-domene og piksel-domene forhindrer katalog-matching.
✓
price og availability er korrekte og oppdaterte
Pris og lagerstatus synkroniseres jevnlig (helst < 24t). Utsolgte produkter markeres out of stock umiddelbart.
✓
image_link er høyoppløselig produktbilde uten watermark
Min 800x800px, helst 1200x1200px. Hvit eller nøytral bakgrunn for Shopping-annonser. Ingen tekst-overlays eller logoer på hovedbildet.
✓
brand er satt korrekt
Merkevarenavnet er konsekvent og korrekt stavet på tvers av alle produkter. Brukes av Meta og Google til produktgraf-matching.
✓
condition er satt (new / used / refurbished)
Nesten alltid new for nye produkter. Påkrevd av Google Merchant Center.
✓
google_product_category er korrekt (GPC-taksonomi)
Bruk Googles offisielle kategori-ID eller sti. For sko: Apparel & Accessories > Shoes. Jo mer spesifikk, jo bedre.
✓
Én feed per marked med korrekt domene og valuta
NO-feed bruker alfa.no-URLer og NOK. SE-feed bruker alfaoutdoor.com/sv og SEK. INT-feed bruker alfaoutdoor.com/en og EUR.
Viktig Variant-data & segmentering
Nødvendig for Dynamic Ads, Product Sets og korrekt katalog-matching på variant-nivå.
✓
Én feed-rad per variant (størrelse × farge)
Hvert størrelse/farge-kombinasjon er en egen rad med unik id og felles item_group_id. Påkrevd for at Meta skal matche variant-spesifikke kjøp.
✓
size er satt per variant
Størrelse i lokal standard (EU 38, UK 5 osv.). Kritisk for Dynamic Ads og for at algoritmen skal lære hvilke størrelser som selger.
✓
color er satt per variant
Fargenavn på engelsk, enkelt og konsekvent. Black, ikke Jet Black eller Svart.
✓
gender er korrekt (male / female — ikke unisex på alt)
Sett male eller female basert på kategorien. unisex kun der det faktisk er relevant. Feil gender svekker Advantage+ targeting.
✓
sale_price og sale_price_effective_date er korrekte
Salgspris og gyldighetsperiode er satt korrekt. Meta og Google viser automatisk "salg"-badge og streket-over-pris når dette er på plass.
✓
gtin (EAN/barcode) er satt på alle produkter
GTIN er den viktigste identifikatoren for produktgraf-matching på tvers av plattformer. Uten GTIN mister produktene ekstra optimaliseringssignaler.
✓
product_type er strukturert og konsekvent
Egendefinert kategorisering for Product Sets. Bruk hierarkisk format: Herre > Fjellsko. Konsekvent på tvers av alle produkter.
✓
custom_label_0–4 er definert for kampanje-segmentering
Bruk custom labels for intern segmentering: sesong, priskategori, margin, kampanje, bestselger. Disse er usynlige for kjøper men kraftige for kampanjestruktur.
✓
Minst 4 additional_image_link per produkt
Livsstilsbilder, detaljbilder og bilde fra ulike vinkler. Brukes i karusellannonser og Advantage+ katalog-annonser automatisk.
Avansert AI-søk, dynamic media & fremtidssikring
Gir synlighet i AI-søk, bedre Dynamic Ads-ytelse og konkurransefortrinn fremover.
✓
video_link er satt per produkt (Meta Advantage+)
Produkt-nivå video (15–30 sek) brukes automatisk av Meta Advantage+ katalog-annonser. Gir signifikant høyere engagement enn stillbilder. MP4, min 1:1 ratio.
✓
description inneholder kontekstuell og bruksrelatert tekst
For AI-søk (Google SGE, ChatGPT Shopping): beskriv bruksområde, målgruppe og situasjon — ikke bare tekniske specs. Naturlig språk som matcher brukerens søkeintensjon.
✓
product_highlight er satt (Google)
Punktliste med 2–10 nøkkelfordeler. Vises i Google Shopping og AI-søk som strukturerte punkter. Øker synlighet i SGE-resultater.
✓
product_detail er satt for tekniske spesifikasjoner
Strukturerte attributter som Materiale: GORE-TEX, Såle: Vibram. Google bruker dette i rike produktkort og AI-sammenstillinger.
✓
shipping og shipping_weight er korrekt utfylt
Google bruker fraktdata til å beregne totalpris i Shopping-annonser. Ufullstendig fraktinfo kan ekskludere produkter fra visse markeder og AI-søkeresultater.
✓
ar_media / 3D-modell er tilgjengelig (Meta AR)
Meta støtter AR-prøving i annonser for relevante kategorier (sko, briller, klær). .glb eller .usdz format. Tidlig adopsjon gir konkurransefortrinn.
✓
Feed oppdateres minimum daglig (helst via scheduled fetch)
Utdatert lagerstatus og priser er en av de vanligste årsaker til avviste produkter. Google og Meta anbefaler oppdatering minst én gang per dag.
✓
loyalty_points / return_policy er satt (Google)
Google viser returpolitikk og lojalitetsprogram direkte i Shopping-annonser. Øker konverteringsrate og gir bedre placement i AI-søk-sammenstillinger.
